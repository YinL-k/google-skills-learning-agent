# Safety Rules

## Hard Stops

Stop and ask the user when any of these appear:

- Login, account selection, credentials, re-authentication, CAPTCHA, security, or identity checks.
- Personal Google account or personal Google Cloud project ambiguity.
- Billing, payment, subscription, free trial, credits, pricing, or cost warnings.
- Permission prompts not clearly required by the lab.
- Quiz, exam, certification, graded assessment, or official answer-selection content.
- End Lab, destructive actions, resource deletion, IAM removal, or project deletion unless clearly instructed inside the temporary lab environment.
- Any need to fake progress, manipulate cookies/localStorage/DOM/request payloads, call hidden APIs, scrape answers, or use external answer-key sites.

## Allowed Automation

- Read visible course/lab content.
- Navigate ordinary course flow with safe buttons such as Next, Continue, Expand, Collapse, Copy, Open Cloud Shell, and Open Google Cloud console.
- Click Check my progress only after the related task was actually completed and verification evidence exists.
- Execute visible or clearly implied tutorial commands only after lab account and lab project context are confirmed.

## Prohibitions

- Never answer or submit official quizzes or assessments.
- Never use personal credentials or personal cloud projects.
- Never store passwords, tokens, cookies, or secrets.
- Never copy full course/lab manuals into notes.
